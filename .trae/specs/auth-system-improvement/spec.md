# PanSou 认证系统改进 - 产品需求文档

## Overview
- **Summary**: 对 PanSou 现有的基于 JWT 的认证系统进行安全改进和功能增强，包括密码哈希存储、更安全的 JWT 密钥生成、完善的认证中间件以及前端认证流程优化。
- **Purpose**: 提升系统安全性，解决当前认证体系中的安全隐患，同时保持系统的灵活性和易用性。
- **Target Users**: 需要使用认证功能的 PanSou 部署用户，特别是私有部署场景下的管理员。

## Goals
- 实现密码哈希存储，替代当前的明文存储方式
- 改进 JWT 密钥生成机制，确保密钥安全性
- 完善认证中间件，确保C端接口不需要认证
- 优化前端认证流程，提升用户体验
- 保持向后兼容性，确保现有部署不受影响

## Non-Goals (Out of Scope)
- 不引入外部认证服务（如 OAuth、LDAP 等）
- 不修改现有的 API 接口结构
- 不增加复杂的用户管理功能（如角色管理、权限细分等）

## Background & Context
- 当前 PanSou 认证系统基于 JWT 实现，支持通过环境变量配置用户账号
- 现有系统存在密码明文存储的安全隐患
- JWT 密钥生成机制不够安全，默认使用时间戳作为密钥
- 认证中间件功能相对简单，缺乏细粒度的权限控制
- 前端认证流程需要进一步优化，提升用户体验

## Functional Requirements
- **FR-1**: 实现密码哈希存储，使用安全的哈希算法（如 bcrypt）处理用户密码
- **FR-2**: 改进 JWT 密钥生成机制，确保密钥的随机性和安全性
- **FR-3**: 完善认证中间件，确保C端接口不需要认证，只对管理接口进行认证
- **FR-4**: 优化前端认证流程，提升用户体验
- **FR-5**: 保持向后兼容性，确保现有部署配置仍然有效

## Non-Functional Requirements
- **NFR-1**: 安全性 - 密码必须使用强哈希算法存储，JWT 密钥必须足够安全
- **NFR-2**: 性能 - 认证过程不应显著增加系统响应时间
- **NFR-3**: 可靠性 - 认证系统必须稳定运行，避免认证失败导致的服务不可用
- **NFR-4**: 可维护性 - 代码结构清晰，易于理解和维护

## Constraints
- **Technical**: 保持与现有 Go 版本和依赖库的兼容性
- **Business**: 最小化对现有部署的影响，确保平滑升级
- **Dependencies**: 可能需要引入 bcrypt 等密码哈希库

## Assumptions
- 现有用户配置格式（user:pass 格式）将保持不变
- 认证系统仍然是可选功能，通过 AUTH_ENABLED 环境变量控制
- JWT 仍然是认证令牌的主要实现方式

## Acceptance Criteria

### AC-1: 密码哈希存储
- **Given**: 管理员通过环境变量配置用户账号
- **When**: 系统启动时加载用户配置
- **Then**: 密码被自动哈希处理并安全存储
- **Verification**: `programmatic` - 检查配置加载代码是否使用 bcrypt 等安全哈希算法

### AC-2: JWT 密钥安全生成
- **Given**: 未设置 AUTH_JWT_SECRET 环境变量
- **When**: 系统启动时
- **Then**: 生成足够安全的随机 JWT 密钥
- **Verification**: `programmatic` - 检查密钥生成代码是否使用密码学安全的随机数生成

### AC-3: 认证中间件完善
- **Given**: 启用认证功能
- **When**: 请求访问需要认证的接口
- **Then**: 中间件正确验证 Token 并处理各种错误情况
- **Verification**: `programmatic` - 编写测试用例验证中间件的各种场景

### AC-4: 前端认证流程优化
- **Given**: 用户登录系统
- **When**: 进行各种操作（如搜索、查看结果等）
- **Then**: 前端正确处理登录状态、Token 过期等情况
- **Verification**: `human-judgment` - 手动测试前端认证流程的各种场景

### AC-5: 向后兼容性
- **Given**: 现有部署使用旧的配置方式
- **When**: 升级到新的认证系统
- **Then**: 系统仍然正常工作，不需要修改现有配置
- **Verification**: `programmatic` - 测试使用旧配置格式的部署是否正常工作
