# PanSou 系统技术架构文档

## 1. 系统概述

PanSou 是一个高性能的网盘资源搜索 API 服务，支持从 Telegram 频道和多个第三方数据源搜索网盘链接。系统采用 Go 语言开发，具有以下核心特性：

- 多源搜索：支持从 Telegram 频道和多个插件数据源搜索
- 智能结果合并：自动去重、排序和分类
- 高性能缓存：两级缓存系统，提升响应速度
- 插件化架构：支持动态扩展数据源
- 并发搜索：高效的并行搜索机制
- 完整的部署配置：支持 Docker 容器化部署

## 3. 核心功能模块

### 3.1 搜索服务 (SearchService)

搜索服务是系统的核心，负责协调多个数据源的搜索操作，包括：

- 并行搜索 Telegram 频道和插件数据源
- 智能合并搜索结果（去重、排序、分类）
- 缓存管理和优化
- 结果过滤和处理

关键功能：
- `Search()`: 主搜索方法，支持多源并行搜索
- `searchTG()`: 搜索 Telegram 频道
- `searchPlugins()`: 搜索插件数据源
- `mergeResultsByType()`: 按网盘类型合并结果
- `sortResultsByTimeAndKeywords()`: 智能排序结果

### 3.2 插件系统

插件系统采用可扩展的架构，支持动态添加新的搜索数据源：

- 插件自动注册机制
- 异步搜索能力
- 统一的插件接口
- 插件优先级管理

目前系统集成了 80+ 个搜索插件，涵盖各种网盘资源来源。

### 3.3 缓存系统

系统实现了增强版两级缓存：

- 内存缓存：快速响应
- 磁盘缓存：持久化存储
- 延迟批量写入：优化磁盘操作
- 智能缓存键生成：确保缓存有效性

### 3.4 API 接口

API 层提供了简洁的 RESTful 接口：

- `/api/search`: 搜索接口（支持 GET 和 POST）
- `/api/auth/*`: 认证相关接口
- `/api/health`: 健康检查接口

## 4. 业务流程

### 4.1 搜索流程

1. **接收请求**：API 层接收搜索请求，支持 GET 和 POST 两种方式
2. **参数处理**：解析并验证搜索参数，设置默认值
3. **缓存检查**：检查是否有缓存结果
4. **并行搜索**：
   - 搜索 Telegram 频道
   - 搜索插件数据源
5. **结果合并**：合并、去重、排序搜索结果
6. **结果分类**：按网盘类型分组
7. **缓存更新**：异步更新缓存
8. **返回响应**：返回标准化的搜索结果

### 4.2 插件搜索流程

1. **插件选择**：根据请求参数选择要使用的插件
2. **并行执行**：使用工作池并行执行插件搜索
3. **结果处理**：处理每个插件的搜索结果
4. **缓存更新**：实时更新缓存
5. **结果合并**：将插件结果与其他来源结果合并

## 6. 系统架构

### 6.1 架构层次

1. **API 层**：处理 HTTP 请求，参数验证，响应格式化
2. **服务层**：业务逻辑处理，搜索协调，结果合并
3. **插件层**：各种数据源的搜索实现
4. **工具层**：缓存、HTTP 客户端、工作池等基础设施

### 6.2 核心数据流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 客户端请求  │────>│   API 层    │────>│  服务层     │────>│  插件层     │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
     ^                      │                      │                      │
     │                      │                      │                      │
     │                      v                      v                      v
     └─────────────────────┌─────────────────────┐─────────────────────┘
                           │      缓存系统       │
                           └─────────────────────┘
```

### 6.3 关键组件

- **Gin 框架**：处理 HTTP 请求和路由
- **工作池**：管理并发搜索任务
- **两级缓存**：内存 + 磁盘缓存
- **插件系统**：动态扩展数据源
- **Docker**：容器化部署

## 7. 部署配置

### 7.1 Docker 部署

**使用 Docker Compose**:

```yaml
version: '3.8'

services:
  pansou:
    image: ghcr.io/fish2018/pansou:latest
    container_name: pansou
    restart: unless-stopped
    ports:
      - "8888:8888"
    environment:
      - PORT=8888
      - CHANNELS=tgsearchers4,...
      - ENABLED_PLUGINS=labi,zhizhen,...
      - CACHE_ENABLED=true
      - CACHE_PATH=/app/cache
      # 其他环境变量...
    volumes:
      - pansou-cache:/app/cache
    networks:
      - pansou-network

volumes:
  pansou-cache:
    name: pansou-cache

networks:
  pansou-network:
    name: pansou-network
```

| 环境变量 | 说明 | 默认值 |

## 8. 插件系统

### 8.1 插件接口

所有插件必须实现以下接口：

```go
type AsyncSearchPlugin interface {
    Name() string            // 插件名称
    Priority() int           // 优先级（1-4，数字越小优先级越高）
    Search(keyword string, ext map[string]interface{}) ([]model.SearchResult, error)  // 搜索方法
    AsyncSearch(keyword string, searchFunc SearchFunc, cacheKey string, ext map[string]interface{}) ([]model.SearchResult, error)  // 异步搜索
    SetMainCacheKey(key string)  // 设置主缓存键
    SetCurrentKeyword(keyword string)  // 设置当前关键词
    SkipServiceFilter() bool  // 是否跳过服务层过滤
}
```

### 8.2 插件注册

插件通过 `init()` 函数自动注册：

```go
func init() {
    plugin.RegisterPlugin(&MyPlugin{})
}
```

### 8.3 插件分类

插件按功能和数据源类型分类，主要包括：
- 网盘搜索插件
- 视频资源插件
- 文档资源插件
- 其他资源插件
